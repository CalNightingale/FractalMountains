Fractal Mountains

Based on this paper by Eli Meir and Ben Haller

Part 1: Making the triangles

We represent the mountain as a collection of vertices. The most efficient way to store them is simply as a 2D array of height values. The x and y coordinates can be derived from the index in the array. The z coordinate is the height value. The triangles are then simply the 3 vertices that make up each triangle. The smallest supported number of points is 4. We use this to create a rombus.
